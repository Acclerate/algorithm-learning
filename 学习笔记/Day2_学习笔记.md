## 💡 重要概念与理解

### 快慢指针的三种应用层次

| 层次 | 题目 | 判断条件 | 难度 |
|------|------|----------|------|
| 基础 | LeetCode 27 | `nums[fast] != val` | Easy |
| 进阶1 | LeetCode 26 | `nums[fast] != nums[fast-1]` | Easy |
| 进阶2 | LeetCode 80 | `nums[fast] != nums[slow-2]` | Medium |

**核心思想演进**:
```
基础: 只要不等于某个值，就保留
进阶1: 不等于前一个元素，说明不重复
进阶2: 不等于slow-2位置的元素，说明没超过2次
```

## 📝 学习过程中的重要问答

### Q1: LeetCode 26 和 LeetCode 27 有什么区别？

**我的理解**:
- LeetCode 27: 移除所有等于 val 的元素（给定值）
- LeetCode 26: 移除重复元素（保留第一个）
- 都用快慢指针，但判断条件不同

**对比**:
```java
// LeetCode 27
if (nums[fast] != val) {
    nums[slow++] = nums[fast];
}

// LeetCode 26
if (fast == 0 || nums[fast] != nums[fast-1]) {
    nums[slow++] = nums[fast];
}
```

### Q2: LeetCode 80 为什么要比较 nums[slow-2]？

**我的困惑**:
为什么不比较 nums[fast-1]？

**老师解答**:
关键：slow 前面的元素都是已确定的有效元素
- slow-2 位置是"当前元素的第2个位置"
- 如果 nums[fast] == nums[slow-2]
  说明已经有2个相同的元素了，不能再加
- 如果 nums[fast] != nums[slow-2]
  说明可以安全添加

**我的理解**:
```
例如 slow=5，前面已确定: [1,1,2,2,3]
slow-2 位置是 2（索引2）
如果要添加新元素 2:
nums[fast] = 2, nums[slow-2] = 2
相等 → 已有2个2了，跳过

如果要添加新元素 3:
nums[fast] = 3, nums[slow-2] = 2
不相等 → 可以添加
```

### Q3: 为什么 slow 和 fast 都从 2 开始？

**我的理解**:
- 前两个元素直接保留（最多出现2次）
- 第1个元素：肯定保留
- 第2个元素：无论是否重复，都保留（出现1或2次都符合要求）
- 从第3个元素（索引2）开始判断

**验证**:
```
[1,1,1,2,2,3]
 ↑↑
 保留这2个

从索引2开始判断
```

## 🎯 今日完成的题目

### LeetCode 26 - 删除有序数组中的重复项

**掌握程度**: ⭐⭐⭐⭐ (85/100分)

**关键代码**:
```java
if (nums == null || nums.length == 0) {
    return 0;
}

int slow = 0;
for (int fast = 0; fast < nums.length; fast++) {
    if (fast == 0 || nums[fast] != nums[fast - 1]) {
        nums[slow++] = nums[fast];
    }
}
return slow;
```

**收获**:
- ✅ 理解去重的判断条件
- ✅ 掌握 `nums[fast] != nums[fast-1]`

**遇到的问题**:
- 拼写错误：`lenght` → `length`
- 语法格式问题：但核心逻辑正确

### LeetCode 80 - 删除有序数组中的重复项 II

**掌握程度**: ⭐⭐⭐⭐⭐ (100/100分！满分！)

**关键代码**:
```java
if (nums == null || nums.length <= 2) {
    return nums.length;
}

int slow = 2;
for (int fast = 2; fast < nums.length; fast++) {
    if (nums[fast] != nums[slow - 2]) {
        nums[slow++] = nums[fast];
    }
}
return slow;
```

**收获**:
- ✅ 理解 `slow-2` 的巧妙设计
- ✅ 掌握快慢指针的进阶应用
- ✅ 理解边界条件处理

**代码完美**:
- 无任何错误
- 逻辑清晰
- 代码规范

## 🆚 三道题目的对比

| 特性 | LeetCode 27 | LeetCode 26 | LeetCode 80 |
|------|-------------|-------------|-------------|
| **难度** | Easy | Easy | Medium |
| **要求** | 移除==val的元素 | 每个元素最多1次 | 每个元素最多2次 |
| **判断条件** | `!=val` | `!=nums[fast-1]` | `!=nums[slow-2]` |
| **slow初始值** | 0 | 0 | 2 |
| **fast初始值** | 0 | 0 | 2 |
| **特殊情况** | 无 | fast==0 | length<=2 |

**核心规律**:
```
随着题目要求变复杂：
- 判断条件从简单到复杂
- 指针初始值需要调整
- 边界情况需要特殊处理
```
